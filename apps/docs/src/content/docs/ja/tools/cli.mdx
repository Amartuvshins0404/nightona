`nightona` コマンドラインツールは、Nightona（デイトナ）のコア機能へのアクセスを提供し、スナップショットの管理や Nightona サンドボックスのライフサイクル管理を行えます。インストール手順は[こちら](/docs/ja/getting-started#setting-up-the-nightona-cli)をご覧ください。

本リファレンスでは、`nightona` コマンドラインツールでサポートされているすべてのコマンドを、その動作の説明および対応フラグとともに一覧しています。`nightona` 実行時に `--help` または `-h` フラグを付けることで、各コマンドごとのドキュメントを参照できます。

## nightona

Nightona CLI

```shell
nightona [flags]
```

**フラグ**
| ロング | ショート | 説明 |
| :--- | :---- | :---------- |
| `--help` | | nightona のヘルプを表示します |
| `--version` | `-v` | Nightona のバージョンを表示します |

## nightona autocomplete

シェル環境に補完スクリプトを追加します

```shell
nightona autocomplete [bash|zsh|fish|powershell] [flags]
```

**フラグ**
| Long | Short | 説明 |
| :--- | :---- | :---------- |
| `--help` | | nightona のヘルプを表示 |

<Aside type="note">
  bash シェルを使用している場合、オートコンプリート機能をフルに利用するには bash-completion がインストールされていることを確認してください。Linux のインストール: `sudo
          apt-get install bash-completion` macOS のインストール: `brew install
          bash-completion`
</Aside>

## nightona sandbox create

新しいサンドボックスを作成

```shell
nightona sandbox create [flags]
```

**フラグ**
| Long | Short | 説明 |
| :--- | :---- | :---------- |
| `--auto-archive` | | 自動アーカイブの間隔（分）（0 は最大の間隔が使用されます） |
| `--auto-delete` | | 自動削除の間隔（分）（負の値で無効、0 は停止後に即時削除） |
| `--auto-stop` | | 自動停止の間隔（分）（0 は無効） |
| `--class` | | サンドボックスのクラス種別（small、medium、large） |
| `--context` | `-c` | ビルドコンテキストに含めるファイルまたはディレクトリ（複数回指定可） |
| `--cpu` | | サンドボックスに割り当てる CPU コア数 |
| `--disk` | | サンドボックスに割り当てるディスク容量（GB） |
| `--dockerfile` | `-f` | サンドボックスのスナップショット用 Dockerfile のパス |
| `--env` | `-e` | 環境変数（形式: KEY=VALUE） |
| `--gpu` | | サンドボックスに割り当てる GPU ユニット |
| `--label` | `-l` | ラベル（形式: KEY=VALUE） |
| `--memory` | | サンドボックスに割り当てるメモリ（MB） |
| `--public` | | サンドボックスを公開アクセス可能にする |
| `--snapshot` | | サンドボックスに使用するスナップショット |
| `--target` | | ターゲット（リージョン）（eu, us） |
| `--user` | | サンドボックスに関連付けるユーザー |
| `--volume` | `-v` | マウントするボリューム（形式: VOLUME\_NAME:MOUNT\_PATH） |
| `--help` | | nightona のヘルプ |
