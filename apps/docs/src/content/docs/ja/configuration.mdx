## 環境変数を設定する

Nightona で認証するには API キーが必要です。API キーは Nightona プラットフォームから取得できます。

1. [Nightona Dashboard](https://app.daytona.io/dashboard/) にアクセスします。
2. API Keys を開きます。
3. **`Create Key`** ボタンをクリックします。
4. **`.env`** ファイルで **`NIGHTONA_API_KEY`** 環境変数を設定し、API キーを追加します。
5. **`.env`** ファイルで **`NIGHTONA_API_URL`** 環境変数を設定し、Nightona の API URL を指定します。

## 構成オプション

Nightona SDK では、Python と TypeScript で `NightonaConfig` クラスを使用して設定を行えます。`NightonaConfig` クラスは次のパラメータを受け取ります。

- `api_key`: Nightona の APIキー
- `api_url`: Nightona の API の URL
- `target`: サンドボックスを作成する Nightona のターゲット（リージョン）

<Tabs syncKey="language">
<TabItem label="Python" icon="seti:python">
```python
from nightona import NightonaConfig

config = NightonaConfig(
    api_key="your-api-key",
    api_url="your-api-url",
    target="us"
)

```
</TabItem>
<TabItem label="TypeScript" icon="seti:typescript">
```typescript
import { NightonaConfig } from '@nightona/sdk';

const config: NightonaConfig = {
    apiKey: "your-api-key",          
    apiUrl: "your-api-url",     
    target: "us"                  
};
```

</TabItem>
</Tabs>

## 環境変数

Nightona SDK は設定に環境変数を利用できます。SDK は次の環境変数を自動的に参照します：

| Variable              | Description                                | Optional |
| --------------------- | ------------------------------------------ | -------- |
| **`NIGHTONA_API_KEY`** | Nightona の APIキー。                        |          |
| **`NIGHTONA_API_URL`** | Nightona API の URL。                        | Yes      |
| **`NIGHTONA_TARGET`**  | サンドボックスを作成する Nightona のターゲット（ターゲット（リージョン））。 | Yes      |

### 環境変数の設定

Nightona SDK は環境変数から設定を読み込みます。以下の方法で設定できます。

- [**`.env`** ファイルを使用](#using-a-env-file)
- [シェル環境を使用](#using-shell-environment)

#### **`.env`** ファイルを使用

プロジェクトのルートディレクトリに `.env` ファイルを作成します：

```bash
NIGHTONA_API_KEY=your-api-key
NIGHTONA_API_URL=https://your-api-url
NIGHTONA_TARGET=us
```

- `NIGHTONA_API_KEY`: Nightona の APIキー。
- `NIGHTONA_API_URL`: Nightona API の URL。
- `NIGHTONA_TARGET`: サンドボックスを作成する Nightona のターゲット（ターゲット（リージョン））。

#### シェル環境を使用

シェルで環境変数を設定します：

<Tabs syncKey="shell">
<TabItem label="Bash/Zsh" icon="seti:shell">
```bash
export NIGHTONA_API_KEY=your-api-key
export NIGHTONA_API_URL=https://your-api-url
```
</TabItem>
<TabItem label="Windows PowerShell" icon="seti:powershell">
```bash
$env:NIGHTONA_API_KEY="your-api-key"
$env:NIGHTONA_API_URL="https://your-api-url"
```
</TabItem>
</Tabs>

## 設定の優先順位

SDK は設定に対して次の優先順位（高い順）を適用します:

1. コード内で明示的に指定された設定
2. 環境変数
3. 設定ファイル
4. デフォルト値
