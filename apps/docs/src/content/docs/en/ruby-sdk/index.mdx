The Nightona Ruby SDK provides a robust interface for programmatically interacting with Nightona Sandboxes.

## Installation

Install the Nightona Ruby SDK using Bundler by adding it to your Gemfile:

```ruby
gem 'nightona'
```

Then run:

```bash
bundle install
```

Or install it directly:

```bash
gem install nightona
```

## Getting Started

Here's a simple example to help you get started with the Nightona Ruby SDK:

```ruby
require 'nightona'

# Initialize the SDK (uses environment variables by default)
nightona = Nightona::Nightona.new

# Create a new sandbox
sandbox = nightona.create

# Execute a command
response = sandbox.process.exec(command: "echo 'Hello, World!'")
puts response.result

# Clean up
nightona.delete(sandbox)
```

## Configuration

The SDK can be configured using environment variables or by passing options to the constructor:

```ruby
require 'nightona'

# Using environment variables (NIGHTONA_API_KEY, NIGHTONA_API_URL, NIGHTONA_TARGET)
nightona = Nightona::Nightona.new

# Using explicit configuration
config = Nightona::Config.new(
  api_key: 'your-api-key',
  api_url: 'https://app.daytona.io/api',
  target: 'us'
)
nightona = Nightona::Nightona.new(config)
```

## Environment Variables

The SDK supports the following environment variables:

| Variable | Description |
|----------|-------------|
| `NIGHTONA_API_KEY` | API key for authentication |
| `NIGHTONA_API_URL` | URL of the Nightona API (defaults to `https://app.daytona.io/api`) |
| `NIGHTONA_TARGET` | Target location for Sandboxes |
| `NIGHTONA_JWT_TOKEN` | JWT token for authentication (alternative to API key) |
| `NIGHTONA_ORGANIZATION_ID` | Organization ID (required when using JWT token) |
